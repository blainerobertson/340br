Installing XAMPP for Mac

Introduction

An ideal situation for web development is to have a local environment for development and testing and a remote production environment for distributing your work to the rest of the world. To create a local environment requires the installation of server software on your own machine. For web development using PHP, this is known as an "AMP" (Apache, MySQL and PHP) stack. This software bundle gives your machine many of the same capabilities as a dedicated server. On a Mac, the bundle discussed in this tutorial is XAMPP (Apache, MariaDB, PHP and PERL) for Macintosh.

Download

Using your web browser go to apachefriends.org. Once there, click the "Download" link in the navigation bar, then scroll down to the list of downloads for OS X. Find and download the latest NON VM version of PHP 7 version of XAMPP (the fourth item in the list from the bottom) for Mac. Save the installation file to your machine.

Installation

  1. Locate and launch the downloaded installation file on your computer.
  2. If presented with the security warning about opening a file downloaded from the Internet, proceed by clicking "Open".
  3. You will need to enter your computer's password for the install to continue.
  4. When the install begins, accept the default settings for the first three screens. Make sure both checkboxes in the components are selected
  5. On the fourth screen, turn off the checkbox asking if you want to learn more about XAMPP on Bitnami, then click "Next".
  6. Continue clicking "Next" until the installation finishes, then click "Finish".

  1. If the installation finished successfully XAMPP will start automatically and launch a browser window with a splash screen.

Configuration

  1. The installer has created a new XAMPP folder in the Applications folder.
  2. It is within this folder that the local environment runs.
  3. When the installer finished the XAMPP application manager was launched (it may not be visible).
  4. Find and go to it in your running applications. Click the center tab to view the Server Manager.
  5. In the control panel click the "MySQL Database" option in the list, then click the "Start" button on the right. The Apache server should already be running.

  1. Make sure that both server indicators turn green to indicate they are running.
  2. If all of these things have happened as explained, XAMPP is installed and running correctly.
  3. If you encounter an error, consult the "Troubleshooting" section below.

The Site Root

Every web server has a "root" or a folder that holds all of the assets of the web site. In XAMPP that folder is found in Applications > XAMPP > htdocs. Please locate it now.

By default, it already has files and folders in it, do not delete anything until you know what they do. You will add your own folders and files to this folder as you do work in this class.

Running and Stopping XAMPP

Running servers on your local machine is a useful tool, but one that can potentially use much of your computer's resources. Therefore, it is important to start XAMPP only when you are developing and shut it down when finished.

Starting XAMPP

Launch XAMPP and go to the Manage Servers tab of the Application Manager. Then click the Apache and MySQL servers and "Start" each.

Stopping XAMPP

When shutting down, be sure to first "Stop All Servers". Then quit XAMPP. If you quit XAMPP before stopping the servers they will continue to run in the background.

Troubleshooting

Use the following troubleshooting items if the Apache or Database Servers will not start.

Turn Off Conflicting Programs

If the Apache server will not start it may be due to other programs running that are using the web port. Check if any of the following are running:

  • Skype — Quit it (don't just close the window).

Check the Web Port

  1. Normally it is desirable to have Apache running on port 80 for web pages. This is the default web port across the Internet. Check to make sure that is the Apache server port for XAMPP.
  2. In the XAMPP manager, click the Apache Web Server item in the list, then click the "Configure" button on the right.

  1. If it is already 80, click "Cancel".
  2. If the port is NOT showing "80", change it to 80. Click "OK", then click the Apache server "Restart" button. You may be asked to enter your computer password to allow this to happen.

Changing the SSL Port

  1. If you receive an error that Apache cannot start because Port 443 is in use by another program, click the "Config" button next to Apache. In the configuration window, click the "Open Config File" button. Click "Yes" to the warning window that will open.
  2. Search in the file for "httpd-ssl.conf". When found, make sure there is no hashtag (#) at the beginning of the line (on the left). If there is, remove it.
  3. Save and close the httpd.conf file.
  4. Find and open the httpd-ssl.conf file. It should be at /Applications/XAMPP/xamppfiles/etc/extra/httpd-ssl.conf.
  5. Find the line that reads ;Listen 443. Change the Port to 442 or 444, save and close the file. Then try starting Apache again.
  6. Once Apache has started, launch a browser and in the address bar type "localhost" and press enter.
  7. Either the XAMPP language splash screen or the XAMPP success screen will appear. If the splash screen, choose your language. In either case, it means XAMPP is now configured.
