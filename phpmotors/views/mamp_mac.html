MAMP for Mac

Introduction

An ideal situation for web development is to have a local environment for development and testing and a remote production environment for distributing your work to the rest of the world. To create a local environment requires the installation of server software on your own machine. For web development using PHP, this is known as an "AMP" (Apache, MySQL and PHP) stack. This software bundle gives your machine many of the same capabilities as a dedicated server. The bundle discussed in this tutorial is MAMP (Apache, MySQL and PHP) for Mac.

This document covers five topics:

  • Download Mamp
  • Configure Mamp
  • Using the Start Page tools
  • Enable Error Display
  • Troubleshooting

Download MAMP

Video Demonstration

This video demonstrates downloading MAMP for Mac.

Using your web browser go to mamp.info. Once there, find and download the latest version of MAMP by clicking the "Free Download" button next to the MAMP icon on the left (Not the MAMP PRO icon).

When the download page opens click the option on the left for Mac. The file will download to your computer.

Installation

  1. Locate and launch the downloaded installation file on your computer.
  2. Accept the license agreement.
  3. Both MAMP and MAMP PRO will be installed, but we will remove MAMP PRO as we do not wish to purchase a license in order to use it at the end of the installation process.
  4. Accept the defaults for the remainder of the installation process.
  5. When done, click "Finish" to close the installer.
  6. I recommend opening the Applications > MAMP folder and creating an alias of MAMP and adding the alias to the dock that can be used to launch MAMP.
  7. Finally, I recommend finding and deleting the MAMP PRO app in the Application folder, leaving only the MAMP application on your computer.

Configuring MAMP

Video Demonstration

Cloud Drive

Prior to setting up MAMP you will need a cloud drive account and the app for your drive should be installed and working on your computer. Google Drive, Dropbox and OneDrive are all examples of cloud drives. By having a cloud drive, your data is automatically backed up to the drive and should your computer hard drive fail, you will not lose your data.

Create the Web Root Folder

In the cloud drive, on you local computer, create a folder named "htdocs". Make sure it is all lower case! This will become the root folder for your web site and where all work for the class will be done.

Starting Apache and the Database servers in MAMP

  1. Launch MAMP using the dock icon or the app found in the Applications > MAMP folder.
  2. The control panel opens and you can see the status of the servers in the top right corner.
  3. Both servers (Apache and MySQL) should NOT start automatically but we will take steps to change this behavior.

Running and Stopping the MAMP Servers

Running servers on your local machine is a useful tool, but one that can potentially use much of your computer's resources. Therefore, it is important to start MAMP only when you are developing and testing using PHP and shut it down when finished. We will change a few setting to make this easier.

Setting the Servers to Start and Stop Automatically

  1. In the MAMP menu, click the "Preferences" menu option.
  2. In the General tab, turn ON the "Start Servers" checkbox.
  3. Turn OFF the "Open WebStart page" checkbox.

Change the Web Ports

  1. Move to the "Ports" tab.
  2. Click the button labeled "Set Web & MySQL ports to 80 & 3306".
  3. These are standard web ports, and we want to get used to using them.
  4. The port numbers should change in the text boxes above the buttons.

Designate the Root Folder

  1. Click the "Web Server" tab.
  2. Click the "Select..." button to the right of the "Document Root" text.
  3. Navigate to the "htdocs" folder in your cloud drive and select it.
  4. When returned to the MAMP preferences window, make sure the path looks correct.
  5. Click "OK" to close the preferences window.

Start the Servers

  1. When returned to the MAMP control panel, click the "Start Servers" button.
  2. You may be asked to provide your password to start the servers, this is necessary because any port below 1024 on a Mac requires authentication to be used.
  3. You may also see a dialog box indicating that you need to grant permissions for helper tools to be installed for the servers to be started and stopped automatically. Grant such permissions.
  4. If everything works, the circles to the right of the Apache and MySQL servers should turn green.
  5. If they fail to turn green and instead turn red, it indicates a problem and you will need to refer to the video (~ 3:35 mark) or troubleshooting section below.

Test That It Worked

Click the "Open WebStart Page" button. A new web page should open in your default browser. If it does, then things are configured and working. We'll explore this start page in the next section.

The Start Page Tools

The start page serves several purposes: 1) it confirms that the servers are running, 2) it provides access to tools in the MAMP environment and 3) it provides access to information about the environment.

phpMyAdmin

phpMyAdmin is a database management tool and the primary tool that we'll access from the Start page. To access it, select "Tools > phpMyAdmin" from the navigation bar.

The tool's window will open. We'll spend much more time learning and using this tool in future lessons. For now there are two things to remember:

  1. Remember how to get to the tool: Start page > Tools menu > phpMyAdmin.
  2. Never, Ever change any of the four existing databases shown on the left of the screen. These are critical to the basic operation of the database server and should be left alone!

Displaying Errors

By default MAMP hides all errors that it encounters. While this is good in a production environment it is not good for our development environment. Therefore, we need to enable the displaying of errors to help us as we develop our application.

  1. Open phpInfo as done earlier.
  2. Scroll down until you find the item "Loaded Configuration File". Copy the path shown to the right of the title.
  3. Open the finder, click the "Go" menu and click "Go to Folder...".
  4. Paste the path into the dialog box that opens. Hit your Enter key.
  5. Open the php.ini file in any text editor.
  6. Do Command + F to open the search tool.
  7. Search for "display_errors".
  8. Click the "Find Next" button to move down through the page until you find the line that says, display_errors = off (the line will not have a semi-colon at the far left and should be about 25% down from the top of the document at or about line 472).
  9. Change the word "off" to "on".
  10. Save and close the file.
  11. Return to the MAMP control panel and stop, then restart the servers.
  12. From now on, if PHP encounters an error, it will report the error to you at run-time.
  13. You should only have to do this once.

Troubleshooting

Use the following troubleshooting items if the Apache or Database servers will not start

Turn Off Conflicting Programs

If the Apache server will not start it may be due to others programs running that are using the web port. Check if any of the following are running:

  • Skype - Quit it.
  • VMWare â€” Use this Error Correction PDF to turn it off or change settings.
  • Conflicting services - Open the "Activity Monitor" tool and look for any services that are using ports 80 or 3306.
  • Go to the "Go > Utilities > Activity Monitor" application on your Mac.
  • Click the Memory tab.
  • Click the Ports column name to organize all running processes by port number.
  • Look down through the list of ports. If you find anything using port 80 or 3306, click on it to select it, then click the "Force a Process to Quit" icon in the top left corner of the window. Then, click the "Quit" option.
  • Sometimes the ports my be running inside of a virtual machine.
  • If no ports are seen running on port 80 or 3306, click the "Process Name" column heading to alphabetize the processes by name.
  • Look for any process starting with "vm", if found, stop it.
  • When done return to the MAMP control panel and attempt to start the servers again.
  • If you continue to have problems, get help.

Conclusion

With MAMP installed and configured you're ready to begin your work. If for any reason you cannot get the servers to start, get help immediately. Don't put it off. Without the servers you will not be able to proceed in our class.
