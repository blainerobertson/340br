Web Development Checklist

For CSE 340, we will use this checklist for each view you develop, as it reinforces "Best Practice" and is also part of the grading of each view (web page).

The Checklist

  1. ☐ The view is valid HTML (version 5 is the default expectation).
  2. ☐ The view is valid CSS (version 3 is the default expectation).
  3. ☐ All CSS is of your own creation. No outside frameworks or libraries are allowed.
  4. ☐ All CSS, HTML and JavaScript are in separate files.
  5. ☐ The view passes WAVE validation with no error or contrast error.
  6. ☐ The view is responsive and renders correctly on all device screens.
  7. ☐ The view uses the available screen real estate on all screen sizes.
  8. ☐ The content of the view is readable, without zooming and without horizontal scrolling, on mobile screens.
  9. ☐ The <title> element contains a reference to the content of the page (similar to or matching the <h1> value), followed by a seperator and the site name.
  10. ☐ The total download weight of all resources in the view is less than 400kb.
  11. ☐ The view contains an "author" meta element.
