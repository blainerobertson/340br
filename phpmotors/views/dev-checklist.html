Frontend Checklist

For CSE 340, we will use this checklist for each view you develop, as it reinforces "Best Practice" and is also part of the grading of each view (web page).

The Checklist

  1. The view is fully operational. All content renders in the browser without broken links, images, or other content abnormalities.
  2. ☐ The view uses valid HTML and valid CSS (HTML version 5 and CSS version 3 are the default expectations).
  3. ☐ All CSS is of your own creation. No outside frameworks or libraries are allowed.
  4. ☐ All CSS, HTML and JavaScript are in separate files.
  5. ☐ The view headings form an outline with no heading levels being skipped.
  6. ☐ The total download weight of all resources in the view is less than 500kb.
  7. ☐ The view is responsive and renders correctly on all device screens.
  8. ☐ The view uses the available screen real estate on all screen sizes without large blank areas.
  9. ☐ The content of the view is readable, without zooming and without horizontal scrolling, on mobile screens.
  10. ☐ The <title> element contains a reference to the content of the page (similar to or matching the <h1> value), followed by a separator and the site name (e.g. Frontend Checklist | Web Backend Development 1).
  11. ☐ The view passes WAVE validation with no error or contrast error.
