Enhancement 8

Introduction

This week's activities saw us make the navigation bar operational. When a classification name is clicked, that classification's vehicles, from the database, are delivered to a view. This enhancement asks you to take the next step - Specific vehicle information delivered to a view.

Overview

In this enhancement you will:

  1. Alter the code for each vehicle being delivered to the classification view so that a link exists for that specific vehicle. When the link is clicked, a "vehicle detail" view is delivered where complete details about that vehicle are displayed.
  2. Style the vehicle information to display clearly and professionally in any size browser window.

Video Overview

This short video demonstrates what the finished enhancement will look like and the functionality it should possess.

Tasks

  1. Alter the buildVehiclesDisplay() function in the functions.php file to add a link(s) that, when clicked, sends two name-value pairs to the vehicles controller. One name-value pair must trigger a new control structure to get information about a vehicle. The second name-value pair must indicate the specific vehicle (Hint: What is identifier that is unique to each inventory item? Use it as the value in the name - value pair). Suggestion: look at how e-commerce sites such as amazon.com do it (typically both the item name and image are clickable).
  2. Add a new control structure to the vehicle controller that will handle the process of getting a specific vehicle's information in order to deliver it to a vehicle detail view. If no vehicle can be found, a message must be set to that effect and the message will be sent to the view instead of the vehicle information.
  3. Build a function in the "vehicles model" to retrieve information about a specific vehicle in inventory and return it to the controller.
  4. Build a custom function in the functions.php file that will take the specific vehicle's information and wrap it up in HTML to deliver to the view, specifically:
    • Use the full-size image, not the thumbnail image.
    • The make, model and price should be prominent in the view.
    • The price must display as U.S. Dollars with appropriate currency symbol and proper place value commas.
    • The description must be present.
    • The other vehicle items must be displayed with appropriate supporting text to make sense to a site client. See this image from hertz car sales as an example:
  5. Build a new view named "vehicle-detail.php" and save it to the view folder using your template file. The view must:
    • Display the vehicle make and model in the title element and in the main content heading.
    • Have the ability to display a message if one exists.
    • Display the vehicle details by echoing the variable containing the vehicle details wrapped up in HTML from your custom function.
    • Be valid HTML5 and use valid CSS3 to display the vehicle information for easy reading and presenting a professional appearance. For large screens, the image and content must sit side-by-side (similar to the example car image). On small screens, the content should stack into a single column.
    • The content must maintain its clarity of presentation in the browser as the browser window grows or shrinks. In other words, it must be responsive.

Test, Test, Test

Be sure to complete all tasks and then test that they work.

  1. Load the phpmotors site in your local development environment.
  2. Click a link in the main navigation bar.
  3. From the classification vehicles that should be presented in the browser click the provided link(s) for any vehicle.
  4. The vehicles's details should be visible in a new view. The details are clear, professional in appearance that can be easily consumed by the human looking at the browser.
  5. Change the screen size to emulate delivery to various device sizes. The content should be responsive and adjust to allow clear presentation regardless of screen size.
  6. Repeat multiple times (clicking on navigation link, then "drilling down" to the vehicle detail) to ensure that the process works with all vehicles.
