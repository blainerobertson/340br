Keeping Things Straight

One of the most difficult tasks for new programmers is keeping data coming from the browser, being stored into variables, then inserted to database straight. While there is no perfect way to do this, there is a simple practice that may help: use the database field name for the same piece of data in all of its locations.

For example, When registering a new client to the clients table, the field in the table to store the first name is "clientFirstname". So, in the form give the input that name:

<input name="clientFirstname" id="clientFirstname" type="text">
Then, in the controller when collecting the value, store it into a variable of the same name: 
$clientFirstname = filter_input(INPUT_POST, 'clientFirstname');
While not foolproof, it does help to keep track of what form field is sending what data, what variable is storing that value and what database field the data should be stored into.

  2. Run the phpmotors main controller.

  4. Click the "Create a New Account" link. The registration view should be delivered to the browser.
  5. Make sure that the URL in both links is pointing to the accounts controller and passing name - value pairs as parameters to deliver the respective views. Under no circumstances should the links point directly to the view!
